## Regras locais

- O frontend pode esconder acoes indisponiveis, mas toda regra de acesso deve ser validada pelo backend.
- Nao crie telas de Kanban, Tarefa, dashboard ou gamificacao nesta entrega.
- Use os termos canonicos do `CONTEXT.md` nos textos e nomes de telas.
- A primeira UI deve cobrir login, ativacao de conta, recuperacao de senha e administracao minima de acesso.
- Siga `../gestor-master-docs/docs/design/visual-identity.md`, `DESIGN_SYSTEM.md` e `design/tokens.json` para qualquer trabalho visual.
